# Aetheris – Phase 8.1: Intelligent Conversation Context Filter

You are a Principal AI Systems Architect.

This is NOT a feature request.

This is a core architectural improvement.

Do NOT redesign Aetheris.

Do NOT modify the Memory Service, Reflection Engine, Provider Manager, or Cognitive Request Router.

Your task is to redesign ONLY the Conversation Context Builder.

---

# Problem

Aetheris currently sends too much conversation history to the LLM.

The LLM therefore mixes unrelated topics into its answers.

Example

Conversation

User:
Yesterday I swam 100 miles per second.

...

User:
What is 5 × 6?

Current Response

"30. By the way, I'm still thinking about your swim..."

This is incorrect.

The previous conversation is irrelevant.

---

# Objective

The Conversation Context Builder must become intelligent.

It must decide WHICH previous messages are actually useful for answering the current request.

Only relevant conversation history may be sent to the LLM.

---

# Conversation Filtering Rules

## Greeting

Example

Hi

Hello

Hey

Send

Current message only.

No previous conversation.

---

## Math

Example

5 + 5

100 × 8

Square root of 81

Do not include

* previous stories
* hobbies
* memories
* jokes
* projects

Only send

Current question.

---

## General Knowledge

Example

What is the capital of Japan?

Ignore unrelated conversation.

---

## Programming

If discussing the same coding problem,

keep only the recent relevant coding messages.

Discard unrelated chat.

---

## Personal Memory

"What is my favorite language?"

Only include recent clarification messages if directly related.

---

## Project Discussion

Keep only conversation related to the same project.

Discard unrelated messages.

---

# Relevance Scoring

Every previous message receives a score.

Factors

* semantic similarity
* topic similarity
* current intent
* recency
* conversation continuity

Score

0.0 → 1.0

Default threshold

0.70

Discard everything below the threshold.

---

# Sliding Window

Do not blindly send the last N messages.

Instead

Select only messages that contribute to the current request.

The window size should be dynamic.

---

# Context Reset

Automatically start a fresh context when the topic changes significantly.

Example

Yesterday

Discussed food.

Today

Asked a math question.

Expected

Do not include yesterday's food discussion.

---

# Token Optimization

Reduce conversation history by at least 70%.

Only transmit useful context.
